Project Overview

Social interest-matching app. Users pick one of 5 categories, fill in a category-specific bio (all questions are individually skippable), and an AI matches them into a small group chat and fires the first message referencing everyone's bios.

Core flow: Select category → Fill bio (skip any question) → AI matches → Group chat with AI icebreaker

Design System

Theme: Dark, mobile-first (max-width 430px), no build tooling.
Fonts (Google Fonts CDN): Righteous (display/headings) · Poppins (body)

CSS Variables (used in every file)

:root {
  --bg:      oklch(10% 0.012 268);   /* near-black */
  --surface: oklch(14% 0.012 268);   /* card background */
  --border:  oklch(22% 0.008 268);
  --text:    oklch(95% 0.005 268);   /* near-white */
  --muted:   oklch(60% 0.005 268);
  --accent:  <per-category color>;
}

Absolute design bans (impeccable rules — never do these):
Gradient text on body copy · Side-stripe borders · Default glassmorphism · Hero-metric template · Identical card grids · Modal as first solution

Key Constraints to Maintain

  1. All mocks must stay self-contained HTML (no JS frameworks, no build step)
  2. Always use the correct per-category --accent OKLCH value for each screen
  3. Skip buttons must actually disable the input (JS .skipped class pattern)
  4. Chips must toggle with aria-pressed + .selected class
  5. Mobile-first: max-width: 430px, min-height: 100dvh
  6. Follow impeccable bans above
  7. touch-action: manipulation on all interactive elements
  8. prefers-reduced-motion support on all animation screens
